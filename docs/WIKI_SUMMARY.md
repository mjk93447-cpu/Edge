## 1. (최신) ver 20 — 다수의 현실 데이터 적용 시 Auto Optimization 문제점 및 업데이트 필요

**요청 사항 요약**

| # | 내용 |
|---|------|
| 1 | **Score 함수**: 곱연산 위주 → 극단 치우침. 곱·합을 조합해 균형 잡힌 최선 탐색, ML 학습 속도·성능 균형으로 score 함수 전반 업데이트. |
| 2 | **최적화 과정**: 단계 전환 시 높은 score가 사라지고 낮은 score만 반복. 높은 score 파라미터를 유지·활용하도록 Fast/Precise/Perfect 각 목적에 맞게 설계, 단계 간 최고점 반드시 이어서 활용. |
| 3 | **이미지 수**: 최대 100장 → **500장**까지 일괄 분석 가능하도록. |
| 4 | **리포트·GUI**: Auto optimization 결과 리포트·파라미터에 **해당 score 값** 함께 표시. Parameter setting에서 Auto 결과 사용 시 **score도 화면에 표시**. |

마무리: 업데이트 후 코드리뷰·문서 최신화, 비일관/개선점 정리. 수정 시 전체 영향 최소화 계획 수립. 필요 시 커밋/푸시·Actions EXE 생성.

### 코드 반영 현황 (위키 1번~4번)

| 위키 항목 | 반영 내용 |
|-----------|-----------|
| **1. Score 함수 균형** | `compute_auto_score`: 곱연산만 쓰던 식을 **가중 합(weighted_sum) 50% + 가중 기하평균(prod_term) 50%** 로 혼합. 한 지표가 0에 가까워도 전체가 0이 되지 않고, 여러 지표에 고르게 반응하도록 변경. |
| **2. 단계별 최고점 유지** | Adaptive 단계에서 **global best** 를 항상 `top_settings` 맨 앞에 두어, 다음 라운드 후보 생성 시 항상 최고점 파라미터를 중심으로 탐색. 단계 전환 후에도 최고점을 잃지 않도록 함. |
| **3. 이미지 500장** | `max_files` 100 → **500**. 파일 선택 다이얼로그/라벨/경고 문구를 "500"으로 통일. |
| **4. 리포트·파라미터·GUI에 score 표시** | (1) 리포트 끝에 `[BEST]` 블록 추가: best_score (raw·display), 주요 파라미터 요약. (2) `auto_optimize_config.json` 에 **best_score** 필드 저장. (3) Auto 완료 시 `auto_done` 메시지에 best_score 전달 → Parameter Settings 아래 **Auto best score: …** 라벨에 표시. |

## 2. AI image Edge detector ver 19 test report and ver 20 update purpose list

| # | 내용 |
|---|------|
| 1 | **Perfect 모드**: Precise 대비 ~5배 시간 허용, 중요 파라미터 2~10배 촘촘 탐색, 동일 score 함수에 고효율 학습 전략 적용. score 함수·기타 코드 변경 없이 Perfect만 추가. |
| 2 | **Score display raw**: 숫자 지수 형태로 표시해 정확히 확인 가능하게. |
| 3 | 검증 후 GitHub 업데이트·실행파일 생성, 권한 부족 시 안내. |

→ **현재 ver20 코드에 반영된 항목**: Perfect 모드, raw 지수 표시.

## 3. 최신버전 ver18 테스트 완료 및 ver 19 업데이트 개선목표

- Score log10이 -14 이하 → **×10¹⁵ 보정** 표시, 학습 알고리즘에는 미반영.
- 그래프 심미 개선(겹침 해소, 간격·얇은 선, 전문가용 테마).
- **x1e9 제거**.
- 전체화면 스크롤 끝까지 내려가도록 하단 여백 추가.
- 문서·README 최종 정리.

→ ver19에서 반영됨.

---

## 4. 최신버전 ver17 테스트 완료 및 ver18 개선 업데이트 목표

- Score 소수 12자리 출력, 로그 스케일·x1e9를 작은 수 관찰에 맞게 수정.
- 1920×1080에서 스크롤 끝까지 내려가도 버튼 보이도록 수정.
- 문서: 개발 과정·시행착오·배경 전부 추적, 영어 작성.

## 5. v16 review and final ver17 update request

- GUI: score 0.0000만 보이는 문제 → 로그 스케일 또는 10억 배 등으로 사용자 확인 가능하게.
- 그래프 확대: 스크롤·드래그로 원하는 구간 확대, 다수 candidate 시 x축 가독성 개선.
- 극단/실사용 시나리오 테스트로 사용성 개선.
- 문서: 개발 노트, 기능별 성숙도 1~100 평가·개선 아이디어, 포멀 개발문서(한/영).

## 6. [ver15 테스트 결과 피드백 및 개선사항 업데이트 가이드]

- 50장 30분 Auto 결과에서도 **내부 침범 > 주름 윤곽 > 선 끊어짐** 발생.
- 파라미터·step 세분화, 신규 필터/기법 조사·추가·검증.
- 복잡 테스트 이미지로 신규 파라미터 검증.
- Score 함수: 시그모이드 외 AI 문헌 기반 수학 기법 도입·검증.
- Progress에 중요 지표 그래프 추가·확대 가능.

## 7. ver14 피드백 및 업데이트

- 스코어 함수: 우선순위(연속성 > 외곽 피팅 > 두께 > 내부 침범), 시그모이드 등 학습 친화 함수.
- ROI 패턴 유사 이미지 부분 최적화.
- GPU 등 연산 활용, Refine/Adaptive 전략·순서 개선.
